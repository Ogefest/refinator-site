# Key concepts

Designing Refinator few ideas is in background:

  • every file is like database with columns and rows. Sometimes only 1 column and 1 row
  • every task is independent, next task as input take return from previous task and push result to another task. Pipeline concept in Linux (opens new window)
  • task may be applied for column and/or tag
  • only plain text files are supported. CSV, logs, SQL dumps - anything in plain text is fine
  • any file size and rows count are supported. See Limitations for technical limits

Think about Refinator like application which allow you to apply multiple simple tasks on a file to get expected result.
